## 1. Основная цель программы

Программа предназначена для **учёта стран с разными типами государственного устройства**:

* **Президентская республика**
* **Парламентская республика**
* **Смешанная республика**

Для каждой страны программа хранит:

* название страны и континента;
* тип государственного устройства;
* специфические данные в зависимости от типа:

  * президент: Ф.И.О. и средний доход;
  * премьер: Ф.И.О. и население;
  * смешанная: количество партий и «число спидоносных».

Пользователь может вводить **несколько стран**, а потом **просматривать введённые записи**.

---

## 2. Структура типов (объектов)

Программа использует **объектно-ориентированную структуру**:

1. `Reg` — базовый объект, содержит общие данные о стране:

   ```pascal
   Country: String[20];  { название страны }
   Land: String[20];     { континент }
   tip: string[20];      { тип государственного устройства }
   nomer: integer;       { порядковый номер записи }
   ```

2. `prez = object(Reg)` — наследник для президентских республик, добавляет:

   ```pascal
   FIO_Prez: String[50];  { имя президента }
   Sr_Doh: Real;          { средний доход }
   Procedure prez1;       { ввод Ф.И.О. }
   Procedure prez2;       { ввод среднего дохода }
   ```

3. `parl = object(Reg)` — наследник для парламентских республик, добавляет:

   ```pascal
   FIO_Prem: String[50];  { имя премьер-министра }
   Nas: LongInt;          { население }
   Procedure parl1;       { ввод Ф.И.О. }
   Procedure parl2;       { ввод населения }
   ```

4. `smes = object(Reg)` — наследник для смешанных республик:

   ```pascal
   Kol: Integer;          { количество партий }
   Chis: LongInt;         { число спидоносных }
   Procedure smes1;       { ввод количества партий }
   Procedure smes2;       { ввод числа спидоносных }
   ```

---

## 3. Ввод данных

* Пользователь нажимает Enter для ввода новой страны, `*` — чтобы завершить ввод.

* Для каждой страны:

  1. Присваивается номер записи (`nomer := k`).
  2. Вводятся базовые поля: название страны, континент.
  3. Пользователь вводит тип государства (`президентская`, `парламентская`, `смешанная`).
  4. В зависимости от типа вызываются соответствующие методы объектов-наследников:

     * `prez1` / `prez2` — президентская республика
     * `parl1` / `parl2` — парламентская республика
     * `smes1` / `smes2` — смешанная республика

* Каждый ввод числового поля проверяется:

  * `val` используется для преобразования строки в число;
  * если введено некорректное значение (не число или отрицательное число), программа просит повторить ввод.

---

## 4. Методы объектов

Все методы объектов предназначены для **пошагового ввода данных**:

* **prez1 / parl1 / smes1** — ввод строковых полей (Ф.И.О., количество партий).
* **prez2 / parl2 / smes2** — ввод числовых полей с проверкой на корректность.

Например, метод `prez2` для президентской республики:

```pascal
repeat
  write('Введите средний доход: ');
  readln(abc);
  val(abc, Sr_Doh, v);            { преобразование строки в число }
  if (v <> 0) or (Sr_Doh < 0) then
    writeln('Ошибка ввода!!!');
until (v = 0) and (Sr_Doh >= 0);
```

* Цикл повторяется до тех пор, пока пользователь не введёт корректное число ≥ 0.

---

## 5. Просмотр записей

* После ввода всех стран программа переходит в цикл **просмотра записей по номеру**.

* Пользователь вводит номер записи. Если номер некорректный — выводится ошибка.

* Для корректного номера выводятся **все поля страны**, включая:

  * номер записи, название, континент, тип;
  * специфические поля в зависимости от типа:

    * президентская: Ф.И.О. президента, средний доход
    * парламентская: Ф.И.О. премьера, население
    * смешанная: количество партий, число спидоносных

* Цикл просмотра продолжается до тех пор, пока пользователь не введёт `*`.

---

## 6. Важные моменты

1. **Проверка корректности ввода**:

   * строковые поля — читаются без ограничений;
   * числовые поля — проверка с `val` и отрицательных значений.

2. **Использование объектов и наследования**:

   * базовые данные хранятся в `Reg`;
   * специфические данные — в наследниках (`prez`, `parl`, `smes`).

3. **Пользовательский интерфейс**:

   * `ReadKey` используется для управления циклом: Enter — новая запись, `*` — завершение.
   * Сообщения об ошибках при некорректном вводе.

4. **Вывод данных**:

   * Можно выводить отдельные записи по номеру;
   * В последней версии программа выводит **все поля полностью**.

---

## 7. Итог

Программа `Region` позволяет:

* вести учёт стран с различными формами государственного устройства;
* вводить детальные сведения о стране;
* просматривать записи по номеру с полным набором данных;
* гарантировать корректность ввода через проверку числовых значений.
